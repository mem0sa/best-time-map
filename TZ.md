Техническое задание: Best Time Map
1. Цель проекта
Создание веб-сервиса для анализа и определения оптимального времени посещения различных локаций и организаций (кафе, магазины, общественные пространства).
Система помогает пользователям выбрать наиболее комфортный день недели и время визита, основываясь на аналитическом расчёте предполагаемой загруженности.

Проект ориентирован на визуализацию аналитических данных и принятие решений пользователем на основе таблицы загруженности.

2. Роли пользователей
Гость (неавторизованный пользователь):
    просмотр интерактивной карты;
    выбор локации путём клика на объект карты;
    получение аналитической информации о загруженности;
    просмотр рекомендаций по лучшему времени посещения.

3. Модели данных (сущности)

Location (Локация)
Описание выбранного пользователем объекта.
Поля:
    name — название локации;
    address — адрес;
    latitude — географическая широта;
    longitude — географическая долгота;
    place_type — тип локации (кафе, магазин, парк и т.д.).

VisitLoadProfile (Профиль загруженности)
Аналитическая модель, отражающая уровень загруженности по времени.
Поля:
    location — связь ForeignKey с моделью Location;
    day_of_week — день недели;
    hour — час;
    load_score — числовой коэффициент загруженности.

4. Ключевой функционал (User Stories)

Выбор локации:
Пользователь кликает на организацию на интерактивной карте.

Передача данных:
Информация о выбранной локации отправляется на сервер для дальнейшей обработки.

Аналитический расчёт:
Система рассчитывает уровень загруженности для каждого часа и дня недели на основе внутреннего score-алгоритма.

Визуализация:
Пользователь получает таблицу загруженности, где:
    по горизонтали отображаются дни недели;
    по вертикали — часы работы локации;
    нерабочие часы отмечены прочерком;
    оптимальное время посещения визуально выделено.

Рекомендация:
Система автоматически определяет лучший день недели и час для посещения выбранной локации.

5. Внешние интеграции и аналитика

Интеграции:
Google Maps JavaScript API — для отображения карты и выбора локаций.
Google Places API — для получения базовой информации о выбранных объектах.

Аналитика:
собственный алгоритм расчёта загруженности на основе:
    дня недели;
    часа;
    типа локации;
    эвристических коэффициентов;

формирование таблицы загруженности (тепловой карты) для наглядного анализа.
